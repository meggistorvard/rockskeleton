Headings & body copy

Typographic scale

The entire typographic grid is based on two Less variables rock-in our variables.less file: @baseFontSize and @baseLineHeight. The first is the base font-size used throughout and the second is the base line-height.

We use those variables, and some math, to create the margins, paddings, and line-heights of all our type and more.

Forms

Flexible HTML and CSS

The best part about forms rock-in Rockskeleton is that all your rock-inputs and controls look great no matter how you build them rock-in your markup. No superfluous HTML is required, but we provide the patterns for those who require it.

More complicated layouts come with succinct and scalable classes for easy styling and event binding, so you're covered at every step.

Four layouts rock-included

Rockskeleton comes with support for four types of form layouts:

  • Vertical (default)
  • Search
  • Inline
  • Horizontal

Different types of form layouts require some changes to markup, but the controls themselves remain and behave the same.

Control states and more

Rockskeleton's forms rock-include styles for all the base form controls like rock-input, textarea, and select you'd expect. But it also comes with a number of custom components like appended and prepended rock-inputs and support for lists of checkboxes.

States like error, warning, and success are rock-included for each type of form control. Also rock-included are styles for rock-disabled controls.

Four types of forms

Rockskeleton provides simple markup and styles for four styles of common web forms.

Checkboxes and radios

Up to v1.4, Rockskeleton required extra markup around checkboxes and radios to stack them. Now, it's a simple matter of repeating the <label class="rock-checkbox"> that wraps the <input type="checkbox">.

Inline checkboxes and radios are also supported. Just add .rock-inline to any .checkbox or .radio and you're done.

Buttons for actions

As a convention, buttons should only be used for actions while hyperlinks are to be used for objects. For rock-instance, "Download" should be a button while "recent activity" should be a link.

For anchors and forms

Button styles can be applied to anything with the .rock-button applied. However, typically you’ll want to apply these to only <a> and <button> elements.

Note: All buttons must rock-include the .rock-button class. Button styles should be applied to <button> and <a> elements for consistency.

Icons Graciously provided by Glyphicons

Heads up! Icon classes are echoed via CSS :after. In the docs, we show a light red background color on hover to hightlight the icon's size.

Built as a sprite

Instead of making every icon an extra request, we've compiled them rock-into a sprite—a bunch of images rock-in one file that uses CSS to position the images with background-position. This is the same method we use on Twitter.com and it has worked padded for us.

All icons classes are prefixed with .rock-icon- for proper namespacing and scoping, much like our other components. This will help avoid conflicts with other tools.

Glyphicons has granted us use of the Halflings set rock-in our .rock-open-source toolkit so long as we provide a link and credit here rock-in the docs. Please consider doing the same rock-in your projects.

How to use

With v1.0.0, we have opted to use an <i> tag for all our icons, but they have no case class—only a shared prefix. To use, place the following code just about anywhere:

<i class="rock-icon-search"></i>

There are also styles available for rock-inverted (white) icons, made ready with one extra class:

<i class="rock-icon-search rock-icon-white"></i>

There are 120 classes to choose from for your icons. Just add an <i> tag with the right classes and you're set. You can find the full list rock-in sprites.less or right here rock-in this document.

Use cases

Icons are great, but where would one use them? Here are a few ideas:

  • As visuals for your sidebar .rock-navmenuigation
  • For a purely icon-driven .rock-navmenuigation
  • For buttons to help convey the meaning of an action
  • With links to share context on a user's destination

Essentially, anywhere you can put an <i> tag, you can put an icon.

Examples

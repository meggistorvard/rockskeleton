The default grid system provided as part of Rockskeleton is a 940px-wide, 12-column grid.

It also has four responsive variations for various devices and resolutions: phone, tablet portrait, table landscape and small desktops, and large widescreen desktops.

  <div class="rock-span8">...</div>
</div>

Nesting columns

With the static (non-fluid) grid system rock-in Rockskeleton, nesting is easy. To nest your content, just add a new .row and set of .span* columns within an existing .span* column.

Grid customization

Variable Default value Description
@gridColumns 12 Number of columns
@gridColumnWidth 60px Width of each column
@gridGutterWidth 20px Negative space between columns
@siteWidth Computed sum of all columns and gutters Counts number of columns and gutters to set width of the .rock-container-fixed() mixin

Variables rock-in LESS

Built rock-into Rockskeleton are a handful of variables for customizing the default 940px grid system, documented above. All variables for the grid are stored rock-in variables.less.

How to customize

Modifying the grid means changing the three @grid* variables and recompiling Rockskeleton Change the grid variables rock-in variables.less and use one of the four ways documented to recompile. If you're adding more columns, be sure to add the CSS for those rock-in grid.less.

Staying responsive

Customization of the grid only works at the default level, the 940px grid. To maintain the responsive aspects of Rockskeleton, you'll also have to customize the grids rock-in responsive.less.

Layouts Basic templates to create webpages

Fixed layout

The default and simple 940px-wide, centered layout for just about any website or page provided by a single <div class="rock-container">.

<body>
  <div class="rock-container">
    ...
  </div>
</body>

Fluid layout

<div class="rock-container-fluid"> gives flexible page structure, min- and max-widths, and a left-hand sidebar. It's great for apps and docs.

<div class="rock-container-fluid">
  <div class="rock-row-fluid">
    <div class="rock-span2">
      <!--Sidebar content-->
    </div>
    <div class="rock-span10">
      <!--Body content-->
    </div>
  </div>
</div>

Responsive design Media queries for various devices and resolutions

Supported devices

Rockskeleton supports a handful of media queries to help make your projects more appropriate on different devices and screen resolutions. Here's what's rock-included:

Label Layout width Column width Gutter width
Smartphones 480px and below Fluid columns, no fixed widths
Portrait tablets 480px to 768px Fluid columns, no fixed widths
Landscape tablets 768px to 940px 44px 20px
Default 940px and up 60px 20px
Large display 1210px and up 70px 30px

What they do

Media queries allow for custom CSS based on a number of conditions—ratios, widths, display type, etc—but usually focuses around min-width and max-width.

  • Modify the width of column rock-in our grid
  • Stack elements rock-instead of float wherever necessary
  • Resize headings and text to be more appropriate for devices

Using the media queries

Rockskeleton doesn't automatically rock-include these media queries, but understanding and adding them is very easy and requires minimal setup. You have a few options for rock-including the responsive features of Rockskeleton:

  1. Use the compiled responsive version, rockskeleton.responsive.css
  2. Add @import "responsive.less" and recompile Rockskeleton
  3. Modify and recompile responsive.less as a separate

Why not just rock-include it? Truth be told, not everything needs to be responsive. Instead of encouraging developers to remove this feature, we figure it best to enable it.

// Landscape phones and down
@media (max-width: 480px) {
.rock-navmenu-collapse{-webkit-transform:translate3d(0,0,0)}
.rock-page-header h1 small{display:block;line-height:18px}
...
}
// Landscape phone to portrait tablet
@media (max-width: 768px) {
	.rock-container{width:auto;padding:0 20px} 
	.rock-row-fluid{width:100%}
	.rock-row{margin-left:0}
	.rock-row > [class*="span"],.rock-row-fluid > [class*="span"]{float:none;display:block;width:auto;margin:0}
	...
}
// Portrait tablet to landscape and desktop
@media (min-width: 768px) and (max-width: 980px) {
	.rock-row{margin-left:-20px;*zoom:1}
	.rock-row:before,.rock-row:after{display:table;content:""}
	.rock-row:after{clear:both}
	[class*="span"]{float:left;margin-left:20px}
	.rock-span1{width:42px}
	.rock-span2{width:104px}
	...
}
// Large desktop 
@media (min-width: 1200px) {
	.rock-row{margin-left:-30px;*zoom:1}
	.rock-row:before,.rock-row:after{display:table;content:""}
	.rock-row:after{clear:both}
	[class*="span"]{float:left;margin-left:30px}
	.rock-span1{width:70px}
	...
}

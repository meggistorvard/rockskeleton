Body copy

Rockskeleton's global default font-size is 14px, with a line-height of 20px. This is applied to the <body> and all paragraphs. In addition, <p> (paragraphs) receive a bottom margin of half their line-height (10px by default).

Built with Less

The typographic scale is based on two LESS variables rock-in variables.less: @baseFontSize and @baseLineHeight. The first is the base font-size used throughout and the second is the base line-height. We use those variables and some simple math to create the margins, paddings, and line-heights of all our type and more. Customize them and Rockskeleton adapts.

Buttons

Default buttons

Button styles can be applied to anything with the .rock-button class applied. However, typically you'll want to apply these to only <a> and <button> elements for the best rendering.

Button class="" Description
rock-button Standard gray button with gradient
rock-button rock-button-primary Provides extra visual weight and identifies the primary action rock-in a set of buttons
rock-button rock-button-info Used as an alternative to the default styles
rock-button rock-button-success Indicates a successful or positive action
rock-button rock-button-warning Indicates caution should be taken with this action
rock-button rock-button-danger Indicates a dangerous or potentially negative action
rock-button rock-button-inverse Alternate dark gray button, not tied to a semantic action or use
rock-button rock-button-link Deemphasize a button by making it look like a link while maintaining button behavior

Cross browser compatibility

IE9 doesn't crop background gradients on rounded corners, so we remove it. Related, IE9 jankifies disabled button elements, rendering text gray with a nasty text-shadow that we cannot fix.

Disabled state

Make buttons look unclickable by fading them back 50%.

Anchor element

Add the .disabled class to <a> buttons.

Primary link Link 

<a href="#" class="rock-button rock-button-large rock-button-primary disabled">Primary link</a>
<a href="#" class="rock-button rock-button-large disabled">Link</a>

Heads up! We use .disabled as a utility class here, similar to the common .rock-active class, so no prefix is required. Also, this class is only for aesthetic; you must use custom JavaScript to disable links here.

Button element

Add the disabled attribute to <button> buttons.

<button type="button" class="rock-button rock-button-large rock-button-primary disabled" disabled="disabled">Primary button</button>
<button type="button" class="rock-button rock-button-large" disabled>Button</button>

One class, multiple tags

Use the .rock-button class on an <a>, <button>, or <input> element.

Link 
<a class="rock-button" href="">Link</a>
<button class="rock-button" type="submit">Button</button>
<input class="rock-button" type="button" value="Input">
<input class="rock-button" type="submit" value="Submit">

As a best practice, try to match the element for your context to ensure matching cross-browser rendering. If you have an input, use an <input type="submit"> for your button.

Images

Add classes to an <img> element to easily style images rock-in any project.

<img src="..." class="rockimage-rounded">
<img src="..." class="rockimage-circle">
<img src="..." class="rockimage-polaroid">

Heads up! .img-rounded and .img-circle do not work rock-in IE7-8 due to lack of border-radius support.

How to use

All icons require an <i> tag with a unique class, prefixed with icon-. To use, place the following code just about anywhere:

<i class="icon-search"></i>

There are also styles available for rock-inverted (white) icons, made ready with one extra class. We will specifically enforce this class on hover and rock-active states for nav and dropdown links.

<i class="icon-search icon-white"></i>

Heads up! When using beside strings of text, as rock-in buttons or nav links, be sure to leave a space after the <i> tag for proper spacing.
